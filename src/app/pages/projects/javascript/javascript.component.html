CryptoNalysis
About

This website came about while i was looking for a project that i could feel passionate about to showcase some of my skills. I didn't ever expect it to end up being as complicated as it was but I felt like I had to go through with it since i started. I had to research multiple cryptocurrency exchange websites and go through their api's to get a general idea of what the data I would be scraping

Mean stack app
  • Mean Stack Front End with backend containing User Auth
  • Authentication via JWT using passportJs
  • Bcrypt Salted/Hashed password storage.
  • MongoDB database using mongoose ORM
  • Email service using NodeEmailer for verification/password reset emails
  • Echarts for charting Day/Price for cryptocurrencies.
  • Angular5 frontend
  • Nebular + Bootstrap for frontend components
  • Angular Service worker for asset/data caching enabling the site to be served fully with cached data while offine.
    Aswell as increasing loading times significantly
Data Api
  • Backend was written in Ruby on Rails then once more In Node+Express as I wanted to maintain my skills in both languages.
  • Connects to multiple api's with cron jobs triggering functions scraping data for approx 1560 crypto currencies
  • Databases: Ruby version: PostgreSQL, Node/Express version: MongoDB
Challenges

One of the bigger challenges was the database design since I knew all of the different cryptocurrency exchange website API's I had in mind would not have consistent data across the board.

While many of these connections went unused, i sat down and thought of all the possible scenarios I could run into long term if I never stopped
developing the site.I didnt want to write a ton of code for my automatic jobs to later find that I had to change the database and code significantly

About

At a certain point, I realized the repetition there was on every new website I ever made, Authentication.I would go into old projects and start copy pasting for a few hours until I glued together something reminiscent of my old projects.It was frustrating, dirty and i'd have to go through the project removing CSS classes hoping i wouldnt miss anything.
I had enough, so I decided to make my own template with the least amount of unneeded dependencies as I could come up with.It had to be robust if I was going to be cloning it, and minimal CSS as I may not want to use bootstrap or material or w.e framework is in flavor that month anymore.
